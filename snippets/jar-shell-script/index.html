A Better Shell Script Wrapper for JAR Files

09 September 2024

  • #linux
  • #java

To run a JAR file, usually you would do:

$ java -jar /path/to/jar

It‚Äôs a bit long to type, so some people (me included) would wrap it in a shell script:

# run-jar
#!/usr/bin/bash
java -jar /path/to/jar

This way, you only have to type ./run-jar, or run-jar if you have the script somewhere in your shell‚Äôs PATH.

A problem with this approach is that you now have two files, the shell script and the JAR file. If you move or delete the JAR file, the script breaks. A better way to do this is to ‚Äúinclude‚Äù the JAR file directly in the shell script:

# Add the shebang (#!) line to specify the script's interpreter
# This will tell the shell to run the script using java -jar instead of the
# usual bash or sh
$ echo '#!/usr/bin/java -jar' > run-jar

# Append the JAR file to the script
$ cat /path/to/jar >> run-jar

# Set the file permission to be executable
$ chmod +x run-jar

# Run the jar file
$ ./run-jar

This way, everything you need to run the program is in a single file, making it simpler to manage.
